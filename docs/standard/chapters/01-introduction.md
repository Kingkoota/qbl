# Chapter 1: Introduction

## 1.1 Purpose

QBL is designed to be the universal quantum programming language — one grammar that expresses quantum algorithms, hardware control, and error correction across all physical implementations and dimensional scales.

## 1.2 Design Philosophy

### 1.2.1 Dimension-First

Every construct in QBL is parameterized by dimension. The language does not privilege d=2 (qubits) over d=13 (qudits) — both are first-class. This reflects the physical reality that many quantum hardware platforms (superconducting transmons, trapped ions, photonic systems) naturally support more than two levels.

### 1.2.2 Algebraic Foundation

QBL's gate set is derived from the Weyl-Heisenberg group, not from an ad-hoc collection of named gates. For prime dimension d:

- The **shift** operator X and **clock** operator Z generate all single-qudit Pauli operations
- The d² **Weyl operators** D(a,b) form a complete orthogonal basis for operator space
- The **Clifford group** (generated by QFT, phase gate, and SUM) maps Paulis to Paulis
- Adding one **non-Clifford gate** (T gate / cubic phase) achieves universality

### 1.2.3 Linear Type Safety

Quantum states cannot be cloned (no-cloning theorem). QBL enforces this at the type level:

- Quantum variables have **linear types** — must be used exactly once
- Measurement **consumes** the quantum variable
- The compiler rejects programs that violate no-cloning

### 1.2.4 Hardware Agnosticism

QBL compiles to multiple backends:

- **Statevector simulation** (any d)
- **OpenQASM 3.0** (d=2, IBM/Google/IonQ hardware)
- **Pulse-level control** (hardware-specific waveforms)
- **Qudit-native hardware** (d=13, trapped ion / transmon higher levels)

## 1.3 Relationship to Existing Languages

| Language | QBL Advantage |
|----------|---------------|
| OpenQASM 3.0 | QBL adds pulse control, qudit support, linear types |
| Q# | QBL is hardware-agnostic, supports d>2 natively |
| Qiskit/Cirq | QBL is a true language with compile-time guarantees |
| Silq | QBL targets real hardware, supports error correction |
| QUA | QBL is open, dimension-agnostic, not vendor-locked |

## 1.4 Versioning

This document describes QBL Standard Version 0.3.0. The version follows semantic versioning:

- **0.x.y**: Pre-1.0 development (current)
- **1.0.0**: First stable standard (target)
- Major version changes may break backward compatibility
- Minor versions add features without breaking existing programs

## 1.5 Notation Conventions

- `d` — dimension of the quantum system (default: 13 for qudits, 2 for qubits)
- `n` — number of qudits in a register
- `ω` — primitive d-th root of unity: ω = e^(2πi/d)
- `|j⟩` — computational basis state (j ∈ {0, 1, ..., d-1})
- `⊗` — tensor product
- `†` — conjugate transpose (adjoint)
- `mod` — modular arithmetic
